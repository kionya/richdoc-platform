# RICH DOC Global Hub — Phase 4 설계서 (의료광고법 전수검수 + i18n 전수)

작성일: 2026-06-09
대상: 모든 환자화면 카피를 대한민국 의료광고법 기준으로 검수·수정하고, 남은 하드코딩 문구를 4언어로 번역하며, 의무 고지(부작용·주의사항·면책·유치업자 안내)를 추가한다.

## 1. 컨텍스트 & 확정 결정

Phase 1~3 완료·배포됨. 사이트는 발견→예약→알림까지 동작하며 가격·후기·등급·예약이 환자에게 노출된다. 마케팅 확대 전 **법적 안전화**가 우선이다(CLAUDE.md 절대원칙 1: 의료광고법 우선).

| 항목 | 결정 |
|---|---|
| 법적 정보(등록번호·인증) | **아직 없음/준비중** → 허위 표기 제거, "준비 중" 표기 또는 비움 |
| 마케팅 카피 | **준수 카피로 재작성** (단정·과장·허위 제거, 사실·서비스 중심) |
| 범위 | **전수검수 + 수정 + i18n 전수 한 번에** |
| 금지표현 회귀 방지 | **금지어 자동 스캔 유틸 + 테스트** 포함 |
| 사전심의 대상 여부 | ⚠️ **검증필요** — 게재 전 의료광고 자율심의기구/법률 확인(본 스펙은 고지·안내까지만) |

### 적용 법규(핵심)
의료법 제56조(거짓·과장·비교·중요정보 누락 금지), 제57조(사전심의), 제27조③(환자 유인·알선 — 외국인은 유치업자 등록 시 예외). 5중 방어(CLAUDE.md §2)의 마지막 게이트로 `medical_compliance_checker` 적용.

## 2. 위반 제거·재작성 (홈 및 전 화면)

### 2.1 제거 대상(허위·정부인증 표방)
- 가짜 인증박스 4종("K-Medical / ISO 9001 / Global Healthcare / Safe Clinic") **삭제**.
- "**Government Verified** Partners" + "보건복지부 **평가기준 준수**" 문구 **삭제**(정부 인증 표방).
- "**Official** Medical Tourism Platform" 배지 → 중립 표현으로 교체.
- 허위 사업자등록번호 `123-45-67890`, 허위 전화 `010-1234-5678` → "**준비 중**" 표기 또는 비움.

### 2.2 재작성 대상(단정·과장·보장)
| 현재 | 준수 재작성(의미) |
|---|---|
| 히어로 "World Class K-Beauty / Safe & Professional" | "K-Beauty & K-Medical — 한국 병원을 한곳에서"(안전 단정 제거) |
| 히어로 배지 "Official Medical Tourism Platform" | "외국인 환자를 위한 K-Beauty·K-Medical 허브" |
| 히어로 문단 "상위 1%...책임지고 케어" | "검증된 한국 병원을 비교하고 통역·픽업·사후관리 **안내**를 받으세요"(상위1%·책임보장 제거) |
| 컨시어지 "Safety Guarantee / 대리 수술 100% 차단" | "안전 정보 / 수술 실명제·CCTV 참관 등 **병원별 안전 시스템 안내**"(100% 제거) |
| 컨시어지 "Tax Refund Support / 즉시 처리" | "세금 환급 안내"(즉시·보장 어조 완화) |
| 인증 섹션 제목 "Government Verified Partners" | "제휴 병원 안내" + 면책(아래 3.4) |
| 비교 "최저가" 강조 | "**게시가 기준** 최저 — 실제 비용은 상담 후 결정"(보장 제거) |

> 실제 4언어 카피 문안은 구현 플랜(메시지 사전)에서 확정하며, `medical_compliance_checker`/`ad-review`로 검수 후 게재.

## 3. 의무 고지·면책 추가

### 3.1 부작용·주의사항 노출 (중요정보 누락 방지)
병원 `cautions`(Phase 1에 저장, 환자 상세 **미노출**) → **상세 페이지에 표기**. 4언어 resolveText. 시술 가격표 인접 배치.

### 3.2 가격 면책
가격이 노출되는 모든 지점(상세 가격표, 비교 표, 카드)에 고지: "표시 가격은 **병원 게시가**이며 실제 비용은 상담 후 결정됩니다."

### 3.3 후기 면책
상세 후기 섹션에 고지: "후기는 **개인적 경험**이며 시술 효과를 보장하지 않습니다."

### 3.4 등급 면책
tier 배지 노출 지점(목록 카드·상세·비교)에 Phase 2의 `Tier.disclaimer`("등급은 플랫폼 제휴관계, 의료적 우위 아님")가 **실제로 보이는지** 확인하고, 안 보이면 배지 근처/툴팁로 노출.

### 3.5 유치업자·심의 고지(푸터)
- "외국인환자 유치업자 등록 **준비 중**" 안내.
- ⚠️ 검증필요 문구: "의료광고 사전심의 대상 여부는 게재 전 확인" — 운영자/법률 확인 전제. (코드엔 중립 고지만; 실제 심의필 표기는 등록·심의 후.)
- 재사용 가능한 `ComplianceNotice` 컴포넌트로 3.2~3.5 면책을 표준화.

## 4. i18n 전수

- 홈의 하드코딩 한국어/영어 고정 카피(히어로·컨시어지 4카드·인증/제휴 섹션·푸터)를 메시지 네임스페이스 **`Home`/`Footer`/`Compliance`**(4언어)로 이전 — **준수 재작성 카피**로.
- 상세/비교/예약의 잔여 고정 문자열(면책·부작용 라벨 포함) 다국어화.
- 메시지 키 4언어 구조 동일(parity).

## 5. 의료광고법 게이트 (5중 방어 마지막 단계)
- 재작성한 카피 사전을 `medical_compliance_checker`(및 `ad-review`)로 **최종 교차검수**. 위반 발견 시 수정 후 재검수(반려 루프).
- 게이트 통과를 Phase 4 Done 조건에 포함.

---

## 6. 금지표현 자동 스캔 (회귀 방지, TDD)
- `lib/compliance/forbidden.ts` 순수 함수 `scanForbidden(text): string[]` — 금지 패턴 목록(예: `100%`, `상위 1%`, `최고`, `보장`, `완치`, `부작용 없`, `Government Verified`, `Official`, `World Class`, `guarantee`, `best`, `#1`)에 매칭되는 표현 반환.
- 테스트: 메시지 사전(`messages/*.json`)에서 **마케팅 네임스페이스만**(`Home`/`Footer`/`Nav`/`Hospitals`/`Tier` 등) 평탄화해 `scanForbidden`에 통과시켜 **금지어 0건**을 강제. 누가 다시 위반 문구를 넣으면 테스트 실패.
- **`Compliance` 네임스페이스(면책 문구)는 스캔 제외** — "효과를 **보장하지 않음**" 같은 정당한 부정형이 금지어 `보장`에 거짓양성으로 걸리는 것을 방지. (스캐너는 부정형까지 판별하지 않으므로 면책 문구는 대상에서 분리한다.)
- ⚠️ 스캔은 **보조장치**(거짓양성·음성 가능). 최종 판단은 §5 게이트.

## 8. 범위 경계 (YAGNI)
안 함: 실제 등록번호·인증 취득(운영자), 사전심의 신청(법률), 새 기능. 콘텐츠·고지·번역·회귀 스캔에 집중. 도메인 콘텐츠(병원이 입력한 `about`/`cautions`)의 개별 위반은 관리자 입력 책임이며, 본 스펙은 **표시 틀·면책·플랫폼 카피**를 다룬다.

## 9. 성공 기준 (Phase 4 Done)
1. 홈/상세/비교/예약의 플랫폼 카피에서 §2 위반(허위 인증·정부표방·100%·상위1%·보장 단정)이 제거된다.
2. 부작용·주의사항(cautions)이 상세에 노출되고, 가격·후기·등급 면책이 표시된다.
3. 모든 플랫폼 카피가 4언어로 렌더되고 메시지 키 parity가 유지된다.
4. 유치업자·심의 고지(준비중)가 푸터에 표기된다.
5. `scanForbidden` 기반 사전 테스트가 금지어 0건으로 통과한다.
6. `medical_compliance_checker` 최종 게이트를 통과(위반 0 또는 수정 완료)한다.
7. `npm test` + `npm run build` 통과, 기존 기능 비회귀.
